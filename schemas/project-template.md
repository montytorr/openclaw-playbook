# Project File Specification

## Template

```markdown
# Project Name
**Status:** active | paused | done
**Started:** YYYY-MM-DD
**Last updated:** YYYY-MM-DD

## Goal
What are we building / doing? (1-3 sentences)

## Plan
1. Step one
2. Step two
3. ...

## TODO
- [ ] Pending task
- [x] Completed task

## Decisions
- YYYY-MM-DD: Decision made and why

## Notes
- Anything worth remembering
```

## Rules
- Create when starting anything multi-step or complex
- Update TODO/status after completing work
- Mark `Status: done` when finished
- Archive to `projects/archive/` when cluttered
- Check active projects at session start

## Naming
- File: `projects/<project-name>.md`
- Lowercase with hyphens
- Descriptive but concise

## Status Values
- `active` — currently being worked on
- `paused` — on hold, will resume
- `done` — completed

## Archive vs Delete
- Archive: project complete but may be referenced later
- Delete: abandoned with no useful learnings
